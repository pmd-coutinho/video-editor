## Skill Loading

Before editing files for a substantial task:
- Run `pnpm dlx @tanstack/intent@latest list` from the workspace root to see available local skills.
- If a listed skill matches the task, run `pnpm dlx @tanstack/intent@latest load <package>#<skill>` before changing files.
- Use the loaded `SKILL.md` guidance while making the change.
- Monorepos: when working across packages, run the skill check from the workspace root and prefer the local skill for the package being changed.
- Multiple matches: prefer the most specific local skill for the package or concern you are changing; load additional skills only when the task spans multiple packages or concerns.

## Project Context

- This project was scaffolded from the TanStack CLI output generated with: `npx @tanstack/cli@latest create my-tanstack-app --agent --package-manager pnpm --tailwind`.
- Follow-up TanStack Intent commands run after scaffolding: `npx @tanstack/intent@latest install` and `npx @tanstack/intent@latest list`.
- Stack: React, TanStack Start, TanStack Router file-based routing, Vite, TypeScript, pnpm, and Tailwind CSS 4. The default TanStack devtools and test toolchain are retained.
- The app intentionally has one blank home route at `src/routes/index.tsx`; do not add product features or integrations unless requested.
- `src/routeTree.gen.ts` is generated by `pnpm generate-routes`; never edit it manually.
- No environment variables are required. Put future local secrets in `.env` or `.env.local`; they are Git-ignored.
- Deploy using the output produced by `pnpm build`. Select a TanStack Start-compatible server or platform adapter when a deployment target is chosen.
- Run `pnpm dev` for local development, `pnpm build` before deployment, and `pnpm test` when tests are added.
- Next steps: customize `src/routes/index.tsx`, add file-based routes under `src/routes`, and configure a deployment adapter only when a target platform is selected.
